**Deterministic project scaffolder from structured JSON specs.**

Vega Forge generates starter project skeletons from declarative specification files. The MVP is fully deterministic, runs entirely locally, and requires no AI model calls - it's a template engine driven by structured configuration.

## Why Vega Forge?

Project scaffolding tools often require interactive prompts, package manager hooks, or network access. Vega Forge takes a different approach:

- **Spec-driven** - project structure is defined in version-controllable JSON
- **Deterministic** - same spec always produces identical output
- **Local-only** - no network calls, no package registries, no model inference
- **Reviewable** - preview the full file tree before writing to disk
- **Composable** - specs can be generated by other tools in the Vega suite

## Features

- **Three built-in templates** - static site, Python CLI, and API service starters
- **Spec preview** - see exactly what files will be generated before writing
- **Deterministic generation** - identical specs produce byte-identical output
- **Automatic package naming** - converts project names to valid Python package identifiers
- **README generation** - each project gets a populated README from the spec description
- **LICENSE placeholder** - reminds teams to add their actual license
- **.gitignore** - includes sensible Python defaults
- **Zero dependencies** - standard library only

## Install

```bash
git clone https://github.com/Vega-Starboard/vega-forge.git
cd vega-forge
python3 -m pip install --user -e .
```

Requires Python 3.10+. No external dependencies.

## Quickstart

```bash
# Preview what a spec would generate
PYTHONPATH=src python3 -m vega_forge preview --spec examples/specs/static-site.json

# Generate a project from a spec
PYTHONPATH=src python3 -m vega_forge new --spec examples/specs/python-cli.json --out build/demo-cli
```

## Commands

### `preview`

Display the file tree that a spec would generate without writing to disk.

```bash
vega-forge preview --spec path/to/spec.json
```

Output: Alphabetically sorted list of file paths that would be created.

### `new`

Generate a project skeleton from a spec file.

```bash
vega-forge new --spec path/to/spec.json --out path/to/output/directory
```

Creates the output directory (and parents) with all generated files.

## Spec Format

Specs are JSON files with a simple structure. See [`docs/spec-format.md`](docs/spec-format.md) for the full specification.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Project name (used for directories, README title, HTML title) |
| `type` | string | Template type: `static-site`, `python-cli`, or `api-service` |
| `description` | string | Project description (used in README and HTML body) |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `features` | array | List of feature strings (documented in generated README for reference) |

### Example Spec

```json
{
  "name": "demo-cli",
  "type": "python-cli",
  "description": "Python CLI starter",
  "features": [
    "argparse",
    "verify script"
  ]
}
```

## Templates

### `static-site`

Generates a single-page HTML/CSS starter.

**Files produced:**
- `index.html` - semantic HTML5 with title, H1, and description
- `styles.css` - responsive layout with system font stack
- `README.md` - project description
- `LICENSE` - placeholder
- `.gitignore` - Python defaults

### `python-cli`

Generates an installable Python CLI starter.

**Files produced:**
- `src/<package>/__main__.py` - entry point with hello-world print
- `scripts/verify.py` - placeholder verification script
- `README.md` - project description
- `LICENSE` - placeholder
- `.gitignore` - Python defaults

Package names are automatically derived from the project name (hyphens converted to underscores).

### `api-service`

Generates a FastAPI-style API skeleton.

**Files produced:**
- `app.py` - FastAPI app with graceful import handling
- `.env.example` - PORT configuration template
- `README.md` - project description
- `LICENSE` - placeholder
- `.gitignore` - Python defaults

The generated `app.py` handles missing FastAPI gracefully - it sets `app = None` if FastAPI isn't installed, allowing the project to be generated and reviewed before dependencies are installed.

## Outputs & Artifacts

- **File tree preview** - stdout listing of all files a spec would produce
- **Generated project** - directory with README, LICENSE, .gitignore, and template-specific source files
- **Confirmation message** - `generated <name> at <path>` on successful generation

## Privacy & Safety

- **Local-only** - all generation happens on your machine
- **No network calls** - zero external requests during preview or generation
- **No telemetry** - no data collection of any kind
- **Deterministic** - output is fully predictable from the input spec

## Project Status

**MVP - v0.1.0 released.** Three template types with deterministic generation, spec preview, and basic file output.

### Known Limitations

- Three built-in templates only - no custom template system yet
- No interactive prompts or spec validation beyond required field checking
- Generated projects are skeletons only - no dependency installation or initialization
- No support for nested directory structures beyond what templates define
